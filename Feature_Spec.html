CHANGELOG

Rules Implementation

Rule 1 — JSI/FFI bridge không được expose raw pointer sang JS/WASM layer.

Chỉ được pass serialized message qua IPC channel. Raw pointer sang JS hoặc WASM là hard blocker. Cụ thể hơn: FFI endpoint không được return Vec<u8>, const u8, haymut u8 trực tiếp. Phải đi qua token protocol tera_buf_acquire → u64 opaque → tera_buf_release. Rule này ngăn GC NativeFinalizer race condition và unauthorized memory access từ WASM layer.

Rule 2 — cold_state.db phải được SQLCipher mã hóa với key derive từ Secure Enclave.

Không hardcode encryption key. Key phải được derive từ Hardware Root of Trust, không phải từ user password hay fixed salt. Nếu cold_state.db migration fail, rebuild từ hot_dag.db là safety net bắt buộc — không abort, không data loss.

Rule 3 — Background Notification Extension phải giới hạn RAM ≤ 20MB.

iOS NSE và Android FCM Service đều phải có Circuit Breaker. Nếu memory usage vượt 20MB, process phải tự terminate gracefully, không để OS kill. Static Memory Arena 10MB pattern được recommend cho NSE. Ghost Push skeleton pattern cho oversized payloads. Rule này prevent OOM-Kill trong NSE/FCM context — crash ở đây là notification loss.

Rule 4 — Mọi WASM .tapp phải run trong Sandbox với PROT_READ-only shared memory.

WASM không được write vào DAG buffer. Sandbox phải có Guard Pages. Mọi write ra ngoài sandbox boundary phải trigger SIGSEGV trap và panic ngay lập tức. Đây là isolation guarantee cho untrusted third-party code.

Rule 5 — Clipboard operation phải đi qua Protected Clipboard Bridge.

Không được gọi OS clipboard API trực tiếp. Bridge phải detect display server (Wayland vs X11 trên Linux) và chọn đúng backend: wl-clipboard cho Wayland native, xclip/xsel cho X11. iOS phải dùng UIPasteboard qua bridge. Rule này prevent clipboard snooping bởi malicious apps.

Rule 6 — ZeroizeOnDrop bắt buộc cho mọi struct giữ plaintext.

Verify bằng Miri test — không chỉ code review. Rule này là client-side enforcement của Core_Spec Rule 2, với yêu cầu CI verification cụ thể hơn.

1. [ARCHITECTURE] [IMPLEMENTATION] Client Data Plane & IPC Bridge

Phạm vi: Tổ chức luồng giao tiếp chéo giữa Rust Core Headless và Native Frontend. Hướng tới nguyên lý Zero-Copy và chia rẽ Control/Data Plane.

1.1 App Size Optimization & Link-Time Optimization

  • 📱💻 Core Biên dịch Tĩnh: Build file Rust văng cờ tối ưu dung lượng opt-level = 'z', bật LTO (lto = true), panic = 'abort'. Gọt sạch khối lượng từ ~30MB xuống < 4MB.
  • 📱 Yêu cầu Strip Symbols (loại bỏ symbol dò lỗi) khi Build Release ra file .ipa hoặc .apk.

1.2 [PERFORMANCE] Dual-Plane Isolation

  • 📱💻 Control Plane: Trao đổi các mệnh lệnh, Metadata nhỏ (Protobuf < 50 bytes) như sự kiện gõ phím, Cổng kết nối MQTT. Luồng này chạy liên tục nhưng siêu nhẹ.
  • 📱💻 Data Plane: Luồng chở File, Video 4K, Vector Embedding. Tuyệt đối không nhồi chung đường Control Plane dồn vào JSON/Base64. Nó đi qua Buffer Zero-Copy riêng.

1.3 [PLATFORM] IPC Zero-Copy Transport Matrix

Nền Tảng Lõi Giao Tiếp Data Plane Throughput (Trung bình) Mô tả
📱 iOS C++ JSI Shared Pointer ~400 MB/s Không đi qua WebView Bridge, bypass luồng JS
📱 Android Dart FFI TypedData ~400 MB/s Ký gửi thẳng vào RAM ảo máy
💻 Tauri (Desktop) SharedArrayBuffer ~500 MB/s COOP+COEP Mode, dùng vòng Ring Buffer cực lớn

1.4 Unidirectional State Sync

  • 💻📱 Lõi Rust không chủ động đẩy (push) chuỗi String dài thòng lọng lên Native UI. Thay vào đó, nó bắn tín hiệu mồi (Control Signal) MERGE_COMPLETE.
  • Native UI nhận lệnh, chủ động kéo (pull) ngược lại dữ liệu theo kiểu Window Viewport (Chỉ bốc 20 tin nhắn đang hiển thị). Chống nghẽn RAM do JSON serialization.

1.5 Crash-Safe Checkpoint (Memory Budget Protocol)

OS Constraints: Ứng dụng di động hay bị giết oan (OOM-Kill, Jetsam).

  • 📱 Memory Pinning (iOS Jetsam): iOS từ chối mlock(). Sống sót bằng cách gom bộ nhớ vào kCFAllocatorMallocZone và bắn madvise(MADV_NOCORE) tránh RAM dump ra file crash.
  • 📱 Sự kiện Chặn đứng: App lắng nghe OS System Hook (applicationWillTerminate trên iOS, onTrimMemory trên Android). Tức thời gọi terachat_core_flush_io() dập WAL SQLite ghi ra ổ cứng trước khi nổ bùm đóng App chết nguồn. Toàn trình dưới 50ms.
  • 📱 RAM Circuit Breaker (CHOKE): Tự động cắt cầu IPC và ngừng nạp Event Buffer khi RAM nền vượt hạn ngạch 8MB trên iOS NSE. Tránh thảm họa Jetsam OOM-Kill triệt để.

2. [ARCHITECTURE] [IMPLEMENTATION] Local Storage & SQLite WAL Compact

Phạm vi: Tổ chức lưu trữ trên máy người dùng, bảo vệ tuổi thọ SSD/Flash, và duy trì lịch sử bất biến.

2.1 Kiến trúc Cơ sở Dữ liệu

  • 📱💻 hot_dag.db: DB nhật ký mạng DAG CRDT (Append-Only). Cắm WAL mode. Cấu hình tắt fsync() ngặt nghèo liên tục để gánh Event nhỏ.
  • 📱💻 cold_state.db: Cơ sở SQLite đóng băng (đã merge phẳng), mã hóa bằng SQLCipher AES-256. Dùng làm View bảng cho UI đọc siêu nhanh.
  • 📱💻 tapp_egress.db: Bể đệm biệt lập (Isolated Queue) dành riêng cho luồng mạng đi ra của các App con (WASM .tapp), bảo hộ Lõi Data chính.

2.2 Foreground Opportunistic VACUUM (Chống Phình WAL)

Bài toán: WAL file SQLite bị phù to quá mức do tần suất Append Event nhắn tin theo nhóm.

  • 📱💻 Khi nhận thấy File WAL hot_dag.db-wal nảy lên quá 50MB (Mobile) / 200MB (Desktop) $\rightarrow$ Cần dọn rác.
  • Tiêu chuẩn Kích hoạt: Nhận diện 2 yếu tố: App đang bật Foreground + Thiết bị đang cắm sạc (Pin > 60%).
  • Thực thi Zero-Downtime: Gọi lệnh ngầm SQLite VACUUM INTO 'hot_dag_tmp.db', gom rác sang bản nháp, sau đó đổi tên cực nhanh bằng chuẩn nguyên tử POSIX Rename (rename()). Không đóng băng Lõi đọc/ghi hiện tại của thiết bị. Khống chế việc ăn mòn NAND Flash.

3. [ARCHITECTURE] [IMPLEMENTATION] Hydration & Sync (Shadow Paging)

Phạm vi: Dưỡng phục Dữ liệu bị thâm hụt sau khi ngủ dậy từ lưới mạng chập chờn hoặc cài lại máy (Zombie Sync).

3.1 Shadow Paging Snapshot Rebuild

  • 📱💻 Thay vì Update trực tiếp lên Data đang làm việc (Gây gián đoạn màn hình đang chat), Lõi lưới tiến hành kéo Chunk lịch sử 2MB/s về.
  • Nạp khối đó vào cold_state_shadow.db tĩnh lặng ở luồng chìm (Background).
  • Sau khi xác nhận Merkle Tree Hash trùng khớp, thực hiện Tráo đổi Database (Atomic Switch). UI tự động chớp Reload nhẹ 50ms. Nếu Tải bị gãy nửa đường (Rớt mạng), shadow.db tự bị xóa rác, DB chính không hấn gì.

3.2 Asymmetric Payload Pruning (Mobile)

  • 📱 Mobile Device không đóng vai trò nòng cốt (như Desktop). Quá trình Cắt Tỉa (Pruning) sẽ chém bỏ bớt các nhánh CRDT thừa / cũ sau 7 ngày, chỉ nhận cục Squash State phẳng phiu. Thiết bị Desktop/VPS (Super Node) gánh 100% History DAG thay cho điện thoại kéo RAM.

4. [ARCHITECTURE] [IMPLEMENTATION] WASM Ecosystem & .tapp Lifecycle

Phạm vi: Trình thông dịch Tiện ích con (.tapp) chạy ở biên (Edge). Gắn liền với Triết lý: Không tin tưởng code bên thứ ba, cấm tuyệt đối System Call.

4.1 Kiến trúc 3 Tầng Phân phối & Thực thi

  • Tầng 1 (Marketplace): Web Marketplace lưu danh mục .tapp đã được ký duyệt (Ed25519 bằng TeraChat Enterprise CA). Admin nhấn "Install".
  • ☁️🗄️ Tầng 2 (Bơm qua Mesh): VPS Control Plane nhận lệnh tải WASM Bytecode về Cluster, sau đó phân phối xuống thiết bị qua mạng Mesh (CRDT Event Log).
  • $> \text{VPS KHÔNG chạy logic .tapp, chỉ cấp một bảng SQL Tùy chọn (Blind Storage) để lưu dữ liệu mã hóa mù.}$
  • Tầng 3 (Thực thi thiết bị): App Mobile/Desktop kéo WASM về. UI Sandbox thực thi 100% logic bằng CPU thiết bị. Không bao giờ lộ Plaintext cho VPS.

4.2 Lách chính sách W^X trên iOS (Interpreter Mode)

OS Constraint: iOS áp đặt chính sách W^X (Write XOR Execute) cực kỳ hà khắc. Việc JIT (vừa ghi bytecode vào RAM vừa chạy) sẽ dẫn tới Crash App lập tức.

  • 📱 Quy chuẩn Interpreter: iOS buộc phải dùng WASM Interpreter (Thông dịch tuyến tính) thay vì JIT. Payload WASM hoạt động như một khối Dữ liệu tĩnh (Data Array) đi qua máy trạng thái ảo mà không phát sinh mã máy động.
  • 📱💻 JSI Memory Isolation: Sử dụng JSI HostObject để chia cách hoàn toàn Vùng nhớ Dữ liệu (Data Heap) và Vùng thực thi tiến trình (Text Segment).

4.3 OPA Policy & Lưới phòng thủ Outbound (Egress Firewall)

  • 📱💻 Lõi Rust thiết lập rào chắn Host Bindings. Mã WASM không bao giờ được phép thọc thẳng vào Network Socket của thiết bị. Mọi luồng API Call fetch() từ trong WASM phải đi qua Rust.
  • 📱💻 TLS Certificate Pinning (HPKP): Loại bỏ domain text whitelist. OPA Policy chứa chuỗi tls_pin: "sha256/abc123...". Khi WASM gọi API ra ngoài mạng, Rust extract Server TLS Public Key đối chiếu. Mismatch $\rightarrow$ Kết liễu kết nối $0\text{ms}$.
  • 📱💻 Byte-Quota Enforcement: Rust chặn mọi lệnh Egress nếu khối lượng byte xuất bến vượt quá hạn mức (payload_bytes > 4096), phòng chống tấn công Data Steganography tẩu tán nhạy cảm.

5. [SECURITY] [IMPLEMENTATION] Security Features & DLP (Data Loss Prevention)

Phạm vi: Các biện pháp chống rò rỉ tại bề mặt thiết bị (Edge Detection) ngay cả khi máy bị chiếm quyền hệ điều hành. Thuộc nhóm quy tắc cứng (Hard-Rule Zero-Tolerance).

5.1 RAM Pinning & Phân rã Bộ nhớ (ZeroizeOnDrop)

  • 💻🖥️ Chống Swap RAM: Trên Desktop/Laptop (Windows/macOS), các vùng nhớ chứa Master Key, Session Key, Decrypted Payload bắt buộc được khóa tĩnh (VirtualLock() / mlock()). HĐH không được phép Swap các vùng RAM này xuống Ổ cứng (Pagefile/Swapfile), ngăn Forensic trích xuất.
  • 📱 Cấm mlock() trên Mobile: iOS/Android cấm mlock(), vi phạm sẽ bị Jetsam-Kill. Thay vào đó, áp dụng mẫu thiết kế Rust ZeroizeOnDrop (RAII). Ngay khoảnh khắc biến Plaintext đi ra khỏi Scope vòng đời hàm, vùng nhớ Heap đó bị nhồi mảng 0x00, đè nát nội dung kể cả khi chưa bị Garbage Collector dọn dẹp.

5.2 Màn che Cửa số Hệ điều hành (Screen Leakage Protection)

  • 📱 iOS (Hardware-level Blur): Đăng ký System Hook UIScreen.capturedDidChangeNotification. Hệ thống giám sát AirPlay, Screen Record, Cắm cáp HDMI. Bất kỳ khi nào cờ isCaptured == true nảy lên, Rust lập tức nảy Event SecurityEvent::ScreenCapture. UI che toàn bộ App bằng UIBlurEffect(.dark).
  • 📱 Android (Kernel Surface Gate): Định nghĩa cờ WindowManager.LayoutParams.FLAG_SECURE ngay tầng khai sinh Activity.onCreate(). Việc này thọc sâu vào Kernel Surface Compositor, mọi nỗ lực chụp màn hình trả về ảnh đen ngòm.
  • 📱💻 Smart Clipboard Controller: Hook API OS Clipboard. Khi người dùng Copy nội dung từ Chat TeraChat, văn bản sẽ bị chèn ẩn Marker RTLO. Mọi luồng Paste văng ra ngoài App khác (Notes, Zalo, Browser) đều bị module Rust đánh trượt, ném về chuỗi rỗng.

5.3 Biometric Screen Lock & Cryptographic PIN Fallback

  • 📱💻 Tôn trọng Hardware Root-of-Trust (Secure Enclave / StrongBox / TPM). Mọi I/O trỏ tới SQLite sẽ bị chặn đứng (Block) nếu cờ Trạng thái Sinh trắc học chưa nhả xanh (kSecAccessControlBiometryCurrentSet).
  • 📱💻 Bàn phím PIN dự phòng áp dụng RAM Zeroization ngay sau mỗi ký tự gõ. Không log phím, không lưu mảng Char array ở tầng UI Framework.

6. [UI] [IMPLEMENTATION] UI Components & Render Flow

Phạm vi: Trải nghiệm phản hồi người dùng siêu tốc, duy trì cơ cấu Glassmorphism 60FPS-120FPS mà không gây cháy máy.

6.1 Viewport Culling & Fast-Scroll 120Hz

  • 📱💻 Cản trở quá tải IPC Bridge khi người dùng cuộn (Scroll) tốc độ siêu biên. Áp dụng cơ chế Lazy UI Sync kết hợp Off-screen Rendering Culling (Bẩm sinh của FlatList / SliverList).
  • Giao diện loại bỏ (Unmount) toàn bộ các mảng Text/Row trôi qua khỏi tầm nhìn. Nhấn chìm lệnh gọi FFI giải mã cục bộ. Đi kèm tín hiệu hủy tác vụ terachat_cancel_task(Message_ID).

6.2 Ảo ảnh Thị giác (Metadata-First Virtual Rendering)

  • 📱💻🖥️ Khi File ảnh hoặc Video 4K (Dung lượng 2GB) được gửi xuống máy, UI không đứng đợi tải. Lõi Rust bóc tách Metadata siêu nhẹ (Zero-Byte Stubs < 500 bytes) + Chuỗi đồ họa Blur-hash (50 bytes).
  • Lớp Render nóng (Hot Render) phát lớp mờ thị giác ngay tại nhịp $0\text{ms}$. Lớp File nhị phân đi đường ngầm giải mã ở Background và "Tráo đổi bóng" (Fade-in) khi sẵn sàng.

6.3 Radar Pulse & Báo hiệu Luồng Offline

  • 📱 Visual Mesh Indication: Khi đứt cáp Internet, hạ tầng P2P Mesh nhậm chức. UI dưới đáy sinh hiệu ứng Radar Pulse quét màu tối (#0F172A).
  • Báo hiệu minh bạch cho người dùng (Executive / System Admin) hiểu rằng lưu lượng dồn ngang (Peer-to-Peer) thay vì luồng dọc (Client-to-Server).

7. [ARCHITECTURE] [IMPLEMENTATION] Storage & E2EE Media Flow

Phạm vi: Trải nghiệm truyền tải Media 4K và luồng file Khổng lồ qua khe hẹp di động (Bypass HTTP cục bộ).

7.1 Native Media DataSource Bridge (No Local HTTP)

  • 📱 Quy tắc cứng: Mobile bắt buộc không được dùng Local HTTP Request 127.0.0.1 để stream video, vì OS sẽ chém chết Socket lúc App chạy nền.
  • 📱 iOS (AVAssetResourceLoaderDelegate): Yêu cầu từ Video Player được gọi qua FFI terachat_media_read(offset, length) chọc thẳng vào Ring Buffer 2MB của Rust. Đọc byte giải mã trên RAM, trả về NSData.
  • 📱 Android (MediaDataSource): ExoPlayer ánh xạ readAt() qua FFI để rải byte từ Rust vào ByteBuffer Dart/Native.

7.2 Chunked Compression & Deduplication

  • 💻🖥️ Thực thi thuật toán Salted MLE (Convergent Encryption). File được băm CAS_Hash = HMAC-SHA256(Static_Dedup_Key, file_content). Hỗ trợ triệt tiêu lưu trữ trùng lặp (Deduplication) lên tới 80% đối với các file công ty dùng chung, mà vẫn bảo toàn E2EE.
  • 📱💻 Cắt gọt khung hình trước khi mã hóa (Pre-Encryption Compression). File JPEG/HEIC được nén giảm 30-80% băng thông tải lên ngay tại thiết bị gốc (Bypass bằng nhãn 100% HD).

8. [ARCHITECTURE] [IMPLEMENTATION] Open-Agent Protocol (OAP) & AI Gateway

Phạm vi: Giao diện cho .tapp tương tác với Mô hình ngôn ngữ (LLM) nội hạt hoặc Đám mây, ẩn danh hóa dữ liệu toàn phần (PII Redaction).

8.1 Stream-Direct Open-Agent Protocol (SD-OAP)

  • 📱💻 API terachat.network.stream_file_to_agent cho phép .tapp ném file trực tiếp cho AI qua Lõi Rust TLS socket. File được Rust kéo từng cục 2MB (Chunk) từ VFS, giải mã On-the-fly và đẩy rớt vào TLS HTTP/2 Stream. Không tồn tại quá trình nuốt trọn 1GB vào RAM.

8.2 Interactive SLM Dual-Masking (ISDM)

  • 📱💻 Lõi Rust vận hành NER (Phân tích thực thể) thu nhỏ (ví dụ ONNX/CoreML) ngay tại thiết bị để tước đoạt PII (Tên, SDT, Email) và thay bằng Alias: [REDACTED_PHONE_1].
  • ☁️ LLM đám mây xử lý logic, trả kết quả về biên máy tính. Lõi Rust hoán đổi Alias (De-tokenization) trở lại chuỗi thật, sau đó triệt tiêu bảng tra cứu bằng lệnh ZeroizeOnDrop trong vòng 100ms. Không bao giờ rời khỏi thiết bị.

8.3 Tainted Agent & LLM Iron Dome

  • 📱💻 Lõi Rust bảo chứng cấu trúc trả về bằng kiểu tham số tĩnh Vec<ASTNode> (AST Sanitizer). Cấm các tệp HTML nội tuyến hoặc chèn Markdown theo dõi (External Image Pixel Tracking).
  • Nếu phát hiện AI phản hồi dính độc (Prompt Injection/XSS), gán cờ TAINTED. Buộc App tạm đình chỉ phiên bản .tapp đó nếu số lần rò rỉ vượt quá 3 lần.
