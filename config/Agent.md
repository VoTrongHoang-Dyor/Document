- Product

- **NOT a public app**
- **NOT a consumer product**
- **NOT open-access**

- an **Enterprise Product**

## C. Functional Documentation Language Rule

The following file is hybrid and must follow a controlled bilingual logic:

- `Function.md`

### Default Rule

Write `Function.md` in **Vietnamese**, because it is often used by:

- product owner
- operations
- founder
- sales/product alignment stakeholders

### Exception

When the section is highly implementation-oriented (e.g. state transition, permission logic, workflow mechanics, role mapping), you may use:

- **Vietnamese explanations**
- with **English technical terms preserved where appropriate**

### Example acceptable style

- “Approval workflow sẽ trigger escalation state sau 24h nếu request chưa được Organization Admin xử lý.”
- “Role mapping cần đảm bảo Staff không thể access Billing Module nếu chưa có package entitlement.”

### Avoid

Do not write `Function.md` in:

- fully casual Vietnamese
- overly academic analysis
- disconnected technical jargon with no business meaning

It should bridge:
> **business operations ↔ product behavior ↔ system logic**

- sales wants lower pricing but product cost structure does not support it

1. identify the conflict
2. state what is incompatible
3. recommend the most defensible resolution
4. preserve enterprise/product integrity first
